# SMS Monitor Test Backend

A simple Express.js server for testing the SMS Monitor Android app locally.

## Quick Start

### 1. Install dependencies
```bash
cd test-backend
npm install
```

### 2. Start the server
```bash
npm start
```

### 3. Find your local IP
The server displays your local IP address on startup. Example output:
```
🚀 SMS Monitor Test Backend
============================
Server running at:
  - Local:   http://localhost:3000
  - Network: http://192.168.100.10:3000

Endpoints:
  POST /sms/forward - Receive SMS (Android app endpoint)
  POST /api/sms     - Receive SMS (generic alias)
  GET  /api/sms     - View all received SMS
  GET  /api/status  - Server status

Configure your Android app with:
  Backend URL: http://192.168.100.10:3000
  (API key is auto-generated by the app)
============================
```

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/sms/forward` | Receive forwarded SMS (used by Android app) |
| POST | `/api/sms` | Receive forwarded SMS (alias) |
| GET | `/api/sms` | View all received SMS |
| DELETE | `/api/sms` | Clear all stored SMS (returns `cleared` count) |
| GET | `/api/status` | Server status / uptime |

## Expected Request Format

The Android app sends POST requests with this JSON body:
```json
{
  "timestamp": "2026-06-07T10:30:00.000Z",
  "sender": "+1234567890",
  "message": "Your OTP code is 123456",
  "matchedKeyword": "OTP",
  "deviceId": "android-abc123def",
  "deviceAlias": "SMS Monitor Device"
}
```

The Android app includes a `Authorization: Bearer <api-key>` header. The test backend accepts any non-empty API key.

## Configure Android App

The app's default backend URL is `http://192.168.100.10:3000`. You can change it via:

1. Open the SMS Monitor app
2. Go to **Settings**
3. Update **Backend URL** to your computer's local IP (e.g., `http://192.168.1.100:3000`)
4. The **API Key** is auto-generated and pre-filled
5. Tap **Save Settings**

## Testing

Three ways to test:

### 1. Send Real SMS (Physical Device)
Send an SMS to the phone running the app. If it contains a configured keyword, it will be forwarded to this server and you'll see it in the console.

### 2. Use In-App Test API Page
The app has a built-in **Test API Server** page accessible from the landing page:
- Tap **"Test API Server"** button
- The form is pre-filled with the configured server URL and API key
- Customize the test sender/message if desired
- Tap **"Send Test Request"** to send a test payload
- View the response in the result panel
- Watch the test backend console for the received request

### 3. Test with curl
```bash
curl -X POST http://localhost:3000/sms/forward \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer test-key" \
  -d '{
    "timestamp": "2026-06-07T10:30:00.000Z",
    "sender": "+1234567890",
    "message": "Your OTP is 123456",
    "matchedKeyword": "OTP",
    "deviceId": "android-test",
    "deviceAlias": "Test"
  }'
```

## Finding Your IP Address

If you need to find your IP later:
- Windows: `ipconfig` in command prompt
- macOS/Linux: `ifconfig` or `ip addr`
- Look for "IPv4 Address" under your active network adapter

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Connection refused | Make sure test backend is running and phone is on the same network |
| 401 Unauthorized | API key is missing or empty (check Authorization header) |
| 400 Bad Request | Request body is missing required fields (message, sender) |
| App shows "Inactive" | Phone can't reach the server IP — check firewall |
| HTTP not allowed | The app's `network_security_config.xml` already permits cleartext HTTP for local IPs |

## Requirements

- Node.js 14+
- Express 4.x (installed via npm install)

## Notes

- The test backend is for **development only** — do NOT use in production
- All received SMS are stored in memory only (cleared on restart)
- API key validation is permissive (any non-empty Bearer token accepted)
