# Feature: Tenant Public URL

**Feature ID:** F-PUBLIC-URL
**Status:** ✅ Sprint 8 — Complete
**Last Updated:** 2026-06-11

---

## Overview

### Description

Each tenant can configure a **Public API URL** — the external address that Android devices use to reach the backend. This URL is embedded in QR codes generated by the web admin, allowing devices to claim themselves against the correct backend endpoint.

### Problem Solved

QR codes currently use the internal Docker address (`http://be:6001`) or the web admin's origin (`http://localhost:6002`), which physical Android devices cannot reach. This feature ensures QR codes always contain a URL the device can actually connect to.

### User Story

> As a tenant admin, I want to configure the public-facing API URL so that QR codes I generate work for devices on my network.

---

## Scope

### In Scope

- Add `publicApiUrl` field to Tenant model
- Auto-detect default value on first access
- Allow admin to override via Settings UI
- Use `publicApiUrl` when generating QR codes
- Fallback chain: DB → ENV → request origin → window.location

### Out of Scope

- Per-device URL overrides
- URL validation/health checks
- Multiple public URLs per tenant

## Auto-Detection Logic

```typescript
function detectPublicApiUrl(): string {
  // 1. Check environment variable (set in Docker/build)
  if (process.env.NEXT_PUBLIC_PUBLIC_API_URL) {
    return process.env.NEXT_PUBLIC_PUBLIC_API_URL;
  }
  
  // 2. Fall back to web admin's origin
  if (typeof window !== 'undefined') {
    return window.location.origin;
  }
  
  // 3. Ultimate fallback (server-side)
  return 'http://localhost:6001';
}
```

## QR Code Generation Change

### Before

```typescript
const qrPayload = `${process.env.NEXT_PUBLIC_API_BASE_URL ?? window.location.origin}/v1/claim?code=${code}`;
// Problem: Uses internal Docker URL or web admin URL
```

### After

```typescript
const publicApiUrl = tenant.publicApiUrl ?? detectPublicApiUrl();
const qrPayload = `${publicApiUrl}/v1/claim?code=${code}`;
// Solution: Uses configured public URL with fallback chain
```
